정규 트랙 — Claude를 동료로 (4주)

Claude에게 우리 팀의 규칙을 가르치고(CLAUDE.md), 능력을 쥐여주고(Skills), 도구와 연결하고(MCP), 여럿을 동시에 부리는(Agent Teams) 핵심 4주. 순서대로 쌓입니다.
4주 · 순서대로 이수 권장 · 매주 실습
1주

CLAUDE.md — 팀의 암묵지를 문서화하라강의안 완비

Claude는 매 대화마다 "처음 만난 동료"라 같은 실수를 반복합니다. CLAUDE.md는 Claude가 프로젝트에 올 때마다 자동으로 읽는 "신입 온보딩 매뉴얼"입니다. 50분 안에 내 프로젝트의 v1을 직접 작성합니다.

  • 왜 매번 같은 실수를 반복하는가 — 자동 참조 메모리의 원리
  • "코드로 알 수 없는 것"만 담기 — 좋은/나쁜 CLAUDE.md 5가지 차이
  • 추상적 다짐 대신 검증 가능한 구체적 행동 기준
  • 실습: /init 으로 내 CLAUDE.md v1 작성 + 금지 규칙 추가
2주

Skills — 반복 작업을 능력 패키지로강의안 완비

같은 체크리스트를 매번 붙여넣고 있다면 Skill로 만들 신호입니다. Slash Command가 "매번 명령"이라면, Skill은 "Claude가 맥락 보고 스스로 꺼내 쓰는 능력"입니다.

  • Slash Command vs Skill — 자동 호출이 만드는 차이
  • SKILL.md 중심의 폴더 구조(references·scripts·templates)
  • 개인 / 프로젝트 / 플러그인 — 어디에 둘 것인가
  • 실습: 내 반복 업무 1개를 개인 Skill로 캡슐화
3주

MCP — Claude를 우리 도구와 연결하라강의안 완비

MCP는 Claude를 우리 드라이브·메일·깃허브·DB와 잇는 표준 어댑터입니다. 핵심은 "연결할 수 있다"가 아니라 "연결해도 되는가" — 권한 설계가 먼저입니다.

  • MCP의 3종 기능 — Tools / Resources / Prompts
  • 도입 5원칙 — 읽기 전용부터, 최소 권한, 승인 절차
  • 신입에게 단계적으로 권한 주듯 안전하게 연결
  • 실습: 읽기 전용 MCP 1개 연결 + 권한 경계 검증
4주

Agent Teams — 다중 에이전트와 팀 도입강의안 완비

여러 Claude를 동시에 쓰는 함정과 정답을 배웁니다. 핵심은 "많이 쓰는 것"이 아니라 "충돌 없이 나누는 것". 마지막으로 우리 팀에 그대로 적용할 4주 파일럿 계획서를 완성합니다.

  • 병렬화의 7단계 프레임워크와 "한 파일은 한 에이전트만"
  • 오케스트레이터의 5가지 검증 항목
  • 팀 도입 규칙 5가지 + "사용 횟수 아닌 결과 품질" 지표
  • 실습: 2~3개 에이전트 병렬 실행 + 4주 파일럿 계획서 작성
