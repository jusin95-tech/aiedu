🧭 심화·특강 트랙 — 주제별 깊이 더하기

정규 과정을 넘어 필요한 주제만 골라 듣는 8개 특강. 문서 품질을 3배로 끌어올리는 법부터 디자인·리서치 자동화, LLM 원리, 자가 개선 에이전트까지.
8강 · 골라 듣기 · 정규 트랙 권장
⭐
특강 01

CLAUDE.md 잘 쓰는 법강의안 완비

정규 1주차의 심화판. CLAUDE.md는 가장 적은 노력으로 가장 큰 효과를 내는 파일입니다. 무시당하지 않는 규칙을 쓰는 실전 원칙을 다룹니다.

  • 200줄 이하로 짧고 구체적이며 검증 가능한 규칙
  • "자기 검증 규칙"으로 품질 2~3배 — 최고 레버리지
  • 비즈니스 도메인 용어 정의와 점진적 확장(실수마다 한 줄)
🎨
특강 02

클로드 디자인 13분 완성

브랜드 사이트·PPT·홍보물을 Claude로 빠르게 디자인합니다. 99%가 놓치는 "디자인 시스템 먼저" 원칙으로 결과물을 일관되게 만듭니다.

  • 디자인 시스템(컬러·타이포·컴포넌트)을 먼저 정의하는 이유
  • PRD(최소 기획서) 기반으로 화면 자동 생성
  • 이 강의 소개 사이트에도 쓰인 브랜딩 일관성 원리
🎧
특강 03

Claude + NotebookLM

내 자료에 근거한 리서치와 오디오·요약 자동화를 Claude와 NotebookLM 조합으로 만듭니다. 출처 기반이라 환각이 적습니다.

  • 업로드한 소스에만 근거해 답하는 리서치 방식
  • 팟캐스트·요약·마인드맵 등 자동 생성
  • nlm CLI로 워크플로 자동화
🧠
특강 04

LLM 작동 원리

언어모델이 답을 만드는 원리(토큰·임베딩·어텐션·생성 전략)를 비개발자 눈높이로 이해합니다. 원리를 알면 프롬프트가 달라집니다.

  • 토큰·임베딩·어텐션이 무엇인가
  • 모델이 "다음 단어"를 고르는 방식
  • 원리를 알면 왜 프롬프트·컨텍스트 설계가 좋아지는가
🌳
특강 05

LLM 나무위키 — 지식망 구축

PARA 폴더와 위키 링크로 "나만의 지식 허브"를 만들고, Claude가 그 지식을 읽고 활용하게 만드는 방법입니다. 이 강의 자료도 그렇게 만들어졌습니다.

  • 투입(raw) → 정리(wiki)로 지식을 축적하는 흐름
  • 양방향 링크로 지식을 연결하는 법
  • LLM이 읽고 꺼내 쓰는 지식 시스템 설계
🎭
특강 06

프로젝트 실전 활용 4가지강의안 완비

같은 프로젝트 기능을 매출 분석가·카피라이터·상담원·비서 네 역할로 변신시킵니다. 핵심은 지침(Instruction)과 지식 베이스(Knowledge Base) 두 축입니다.

  • 분석가: 원가표를 넣어 매출이 아닌 "진짜 이익"을 본다
  • 카피라이터: 잘 쓴 문서 2~3개로 우리 회사 문체 재현
  • 상담원·비서: 규정집으로 정확 응대, 메모리로 장기 작업 유지
🦾
특강 07

Hermes Agent 활용 사례

오픈소스 에이전트(Hermes)를 실제 업무에 적용한 사례를 살펴봅니다. 상용 도구 없이도 자율 에이전트를 구성하는 감을 잡습니다.

  • Hermes 에이전트가 무엇이고 어디에 쓰나
  • 실전 활용 사례로 보는 자율 에이전트 패턴
  • 내 업무에 적용할 아이디어 도출
🔄
특강 08

헤르메스 폐쇄 학습 루프

작업 결과를 다시 학습 데이터로 환류시켜, 같은 작업을 반복할수록 더 잘하게 만드는 자가 개선 구조입니다. 이 볼트에도 실제로 적용돼 있습니다.

  • 교정 누적 — 같은 실수를 두 번 하지 않게
  • 스킬 자동 생성·자가 개선
  • 메모리 큐레이션으로 맥락을 지속 축적
